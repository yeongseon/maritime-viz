# 문제 정의

## 기존 방식의 한계

기존 해운물류 정보는 주로 **표, 지도, 2D 대시보드** 중심으로 제공되고 있습니다.
하지만 해운물류는 단순 위치 정보만으로 이해하기 어려운 복잡한 도메인입니다.

예를 들어 다음과 같은 문제들이 동시에 얽혀 있습니다.

- 선박은 항로를 따라 이동하며 특정 항만에 입항한다
- 항만 내부에는 터미널, 선석, 야드, 게이트, 창고가 존재한다
- 컨테이너는 선박, 야드, 트럭, 철도, 창고 사이를 이동한다
- 지연, 혼잡, 체선, 체화, 탄소배출, 비용이 서로 연쇄적으로 영향을 준다

## 기존 방식의 4가지 한계

!!! warning "한계 1: 관계 파악의 어려움"
    객체 간 관계를 직관적으로 파악하기 어렵습니다.
    예: "이 선박이 왜 대기 중인가?" → 표/지도에서는 답이 보이지 않음

!!! warning "한계 2: 병목 위치 식별 어려움"
    병목 발생 위치와 원인을 한눈에 이해하기 어렵습니다.
    예: 게이트 대기열의 원인이 야드 포화인지, 트럭 부족인지 분간 불가

!!! warning "한계 3: 통합적 관점 부재"
    지연, 혼잡, 탄소 문제를 **통합적으로** 보기 어렵습니다.
    각각이 별도 대시보드에 표시되어 인과관계를 추론하기 어려움

!!! warning "한계 4: 데이터 폭증의 역설"
    데이터가 많아질수록 오히려 이해가 더 어려워집니다.
    필터링/검색만으로는 의미 있는 패턴 발견 불가

## 본 프로젝트의 접근

| 한계 | 해결 방식 |
|---|---|
| 관계 파악 어려움 | **온톨로지 기반 관계 모델 + 클릭 시 3D 관계 라인 시각화** |
| 병목 식별 어려움 | **Congestion 오버레이 + 야드 적치율 높이 시각화** |
| 통합 관점 부재 | **단일 화면에서 오버레이 모드 전환** (Default / Congestion / Delay / Carbon) |
| 데이터 폭증 역설 | **선택 기반 점진적 탐색** (선택한 객체 중심으로 관련 정보만 표시) |

## 결과

복잡한 물류 흐름을 **계층적이고 의미적인 3D 공간**에 배치함으로써,
단순한 위치 정보를 넘어 **"왜 이런 상태인가?"** 라는 질문에 답할 수 있는 시각화를 제공합니다.
