# 오버레이 모드

오버레이 모드는 동일한 3D 항만 데이터를 **4가지 다른 관점**에서 볼 수 있게 합니다.

## 1. Default 모드 🌐

**목적**: 항만 전체의 자연스러운 모습 파악

| 시각 변화 | 비고 |
|---|---|
| 터미널 | 기본 회색 (`#334155`) |
| 야드 | 기본 회색, 높이만 적치율 반영 |
| 게이트 | Status 색상 (open/congested/closed) |
| 선박 | Status 색상 |
| 이벤트 마커 | Critical 이벤트만 표시 |
| CO2 plume | 표시 안 함 |

→ 가장 처음 사용자가 보게 되는 화면. 인지 부담을 최소화합니다.

## 2. Congestion 모드 🚧

**목적**: 항만 운영의 병목 구간을 한눈에 파악

| 시각 변화 | 비고 |
|---|---|
| 터미널 색상 | `congestionLevel`에 따라 변경 (low=녹색, high=주황, critical=빨강) |
| 야드 상단 오버레이 | 적치율 색상으로 강조 (>85% 빨강) |
| 게이트 펄스 | 혼잡 게이트는 펄스 sphere 표시 |
| 메트릭 강조 | StatusBar의 Yard Avg가 강조됨 |

### 사용 사례

> "지금 울산항에서 가장 혼잡한 곳은 어디인가?"
> → Congestion 모드 켜면 즉시 빨간색 영역 확인

## 3. Delay 모드 ⏱

**목적**: 지연·혼잡 이벤트의 위치와 영향 범위 식별

| 시각 변화 | 비고 |
|---|---|
| 이벤트 마커 | `delay` + `congestion` 타입 모두 표시 |
| 마커 색상 | severity에 따라 빨강/주황 |
| 메트릭 강조 | StatusBar의 Alerts가 강조됨 |
| 관련 객체 | 마커가 떠 있는 객체를 클릭하면 영향 범위 라인 표시 |

### 사용 사례

> "오늘 어떤 선박/터미널이 지연 중인가?"
> → Delay 모드 → 마커 위치 확인 → 마커 호버로 원인 확인

## 4. Carbon 모드 🌿

**목적**: 탄소 배출 hotspot 및 idle emission 식별

| 시각 변화 | 비고 |
|---|---|
| 선박 위 plume | `co2EmissionRate > 2.5`인 선박에 빨간 반투명 sphere |
| Plume 크기 | 배출률에 비례 |
| 이벤트 마커 | `emission_alert` 타입만 표시 |
| 라벨 | 호버/선택 시 CO2 t/hr 추가 표시 |
| 메트릭 강조 | StatusBar의 CO2 Total이 강조됨 |

### 사용 사례

> "대기 중인 선박이 탄소를 얼마나 발생시키고 있나?"
> → Carbon 모드 → `HMM Promise` (waiting) 위 plume 확인 → 클릭 → idle emission 이벤트 확인

## 모드 전환 디자인

- 컨트롤 패널의 한 줄에 4개 버튼 배치
- 단일 선택 (하나만 활성화 가능)
- 활성 버튼은 파란 테두리 + 파란 배경
- 모드 전환 시 색상은 `lerp`로 부드럽게 전환 (즉각 변경 X)

## 향후 확장 (로드맵)

- **Time Mode** — 시간 흐름에 따른 상태 변화 시뮬레이션
- **Cost Mode** — 운영 비용 hotspot
- **Network Mode** — 글로벌 항로 네트워크 표시
