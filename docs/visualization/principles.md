# 시각화 원칙

## 기본 철학

3D는 단순히 **예쁘게 보이기 위한 요소**가 아니라, **구조와 관계를 공간적으로 이해시키는 장치**로 사용합니다.

> "단순 항만을 3D로 그리는 것"이 아니라 **"의미가 있는 3D 시각화"**

## 5가지 설계 원칙

### 1. 의미 우선 (Semantic-First)

모든 시각 요소는 도메인 의미를 반영합니다.
색상, 크기, 높이, 애니메이션 모두 임의로 결정되지 않으며 데이터와 1:1 대응됩니다.

| 시각 요소 | 의미 |
|---|---|
| 야드 블록 **높이** | 적치율 (0~1 → 0.2~1.7m) |
| 터미널 **색상** | 혼잡도 (low/medium/high/critical) |
| 게이트 **펄스 애니메이션** | 혼잡 상태 |
| 선박 **위치 흔들림** | 대기/접근 상태 |

### 2. 점진적 노출 (Progressive Disclosure)

기본 화면은 항만 전체 구조만 보여주고, **사용자가 객체를 선택하면** 해당 객체의 관계와 상세 정보가 점진적으로 드러납니다.

```
초기 화면: 항만 전체 구조 (개요)
  ↓ 객체 클릭
선택 상태: 관계 라인 + 상세 패널 + 연관 이벤트
  ↓ 관련 객체 클릭
탐색 상태: 새로운 객체 중심으로 컨텍스트 전환
```

### 3. 다중 관점 (Multi-Perspective)

동일한 데이터를 **여러 관점에서** 볼 수 있어야 합니다.

- **Default 모드**: 자연스러운 항만 모습
- **Congestion 모드**: 혼잡도 중심 색상 매핑
- **Delay 모드**: 지연 이벤트 강조
- **Carbon 모드**: CO2 배출량 중심

### 4. 관계 가시화 (Relation Visibility)

객체 간 관계는 **숨겨진 연결**이 아니라 **시각적 경로**로 표현됩니다.
선택된 객체와 관련 객체 사이에 Bezier curve 기반 3D 라인을 그려 관계를 명시합니다.

### 5. 두 가지 뷰 (Dual View)

- **3D 디지털트윈 뷰** (기본) — 공간적 직관성 강조
- **Knowledge Graph 뷰** (전환) — 의미 관계 강조

각 뷰는 같은 선택 상태를 공유하므로, 한쪽에서 선택한 객체가 다른쪽에도 반영됩니다.

## 안티 패턴 (피해야 할 것)

!!! danger "❌ 단순 미려한 3D"
    데이터와 무관한 화려한 효과는 사용자의 의사결정을 방해합니다.

!!! danger "❌ 정보 과부하"
    모든 객체의 모든 정보를 동시에 표시하지 않습니다. 호버/선택 시점에 점진적으로 노출합니다.

!!! danger "❌ 색상 남용"
    색상은 의미 매핑에만 사용하며, 장식적 색상은 배경/조명 수준에서 절제합니다.
